# Experiment Analysis Skill

每次实验结束后，生成结构化的分析报告。

## 触发条件

- 实验完成后
- 用户要求分析实验结果
- 检查 Modal logs 发现实验已完成

## 执行步骤

1. **收集数据**
   - 从 Modal logs 获取最终指标
   - 检查 checkpoint 目录
   - 获取训练过程指标（如有 JSON 输出）

2. **对比 Baseline**
   - 参考 `BASELINE_CONFIG.md` 的基准值
   - 计算相对差异百分比

3. **分析过程**
   - 检查 Loss 曲线趋势
   - 识别异常（loss spike、梯度爆炸等）
   - 分析是否过拟合

4. **提炼 Insights**
   - 总结成功/失败原因
   - 识别新发现
   - 提出后续实验方向

5. **更新文档**
   - 添加到 `docs/experiments/EXPERIMENTS.md`
   - 如有重大发现，更新 `docs/analysis/` 目录

## 判定标准

| BPB 差异 | 判定 |
|----------|------|
| < -1% | 显著提升 ✅✅ |
| -1% ~ 0% | 小幅提升 ✅ |
| 0% ~ +1% | 持平 ➖ |
| +1% ~ +5% | 小幅下降 ⚠️ |
| > +5% | 显著下降 ❌ |
| > +50% | 实验失败 ❌❌ |

## 输出位置

- 简要结果：直接回复用户
- 完整报告：`docs/experiments/reports/YYYY-MM-DD_[实验名].md`
- 更新汇总：`docs/experiments/EXPERIMENTS.md`
