## 待尝试清单

### 🥇 高优先级

#### 1. Muon Optimizer
- **为什么要试**：专为 transformer 设计，榜首在用，可能比 AdamW 更高效
- **当时状态**：我们用 AdamW，BPB 0.986
- **预估收益**：中-高
- **尝试前问**：
  - [ ] 当前优化器是瓶颈吗？
  - [ ] 训练 loss 还在下降吗？如果已经收敛，换优化器可能没用

#### 2. EMA (Exponential Moving Average)
- **为什么要试**：权重平均，更稳定的最终权重，榜首用 decay=0.997
- **当时状态**：我们直接用最后一步的权重
- **预估收益**：低-中（可能 1-2%）
- **尝试前问**：
  - [ ] 训练是否有波动？如果很稳定，EMA 帮助不大
  - [ ] 实现成本低，可以顺手加

---

### 🥈 中优先级

#### 3. BigramHash
- **为什么要试**：显式 bigram 信息，4096 buckets, dim=128，榜首在用
- **当时状态**：我们只有 token embedding，没有 bigram 信息
- **预估收益**：中
- **尝试前问**：
  - [ ] 模型是否在局部模式上表现差？（可以做错误分析）
  - [ ] 会增加参数量，是否还在 16MB 限制内？

#### 4. SmearGate
- **为什么要试**：学习与前一个 token 的混合，增强局部上下文
- **当时状态**：每个位置独立处理
- **预估收益**：低-中
- **尝试前问**：
  - [ ] 类似 BigramHash，如果那个有效，这个可能冗余
  - [ ] 实现简单，成本低

#### 5. U-Net Skip Connections
- **为什么要试**：对称层之间跳跃连接，改善梯度流
- **当时状态**：标准 transformer，无跳跃连接
- **预估收益**：中
- **尝试前问**：
  - [ ] 深层网络更需要，我们 12 层可能收益有限
  - [ ] 如果增加到 16+ 层，这个更重要

---

### 🥉 低优先级

#### 6. 正交初始化 (Orthogonal Init)
- **为什么要试**：更好的训练起点，保持梯度范数
- **当时状态**：用 Kaiming 初始化
- **预估收益**：低（可能 0.5-1%）
- **尝试前问**：
  - [ ] 训练开始阶段是否有问题？
  - [ ] 如果训练已经很稳定，收益可能很小

#### 7. LeakyReLU(0.5)² vs (0.01)²
- **为什么要试**：榜首用 0.5，我们用 0.01
- **当时状态**：LeakyReLU(0.01)²
- **预估收益**：低（-0.0015 bpb）
- **尝试前问**：
  - [ ] 这是一行代码改动，几乎无成本
  - [ ] 可以顺手试

#### 8. Partial RoPE (16/64)
- **为什么要试**：只对部分维度应用 RoPE
- **当时状态**：全部维度都用 RoPE
- **预估收益**：未知
- **尝试前问**：
  - [ ] 需要研究原理，为什么部分 RoPE 更好？

#### 9. VRL (Value Residual Learning)
- **为什么要试**：layer 0 的 V 通过 sigmoid gate 共享
- **当时状态**：每层独立的 V
- **预估收益**：未知
- **尝试前问**：
  - [ ] 需要读论文 arxiv:2410.17897
  - [ ] 理解原理后再决定

#### 10. GPTQ (Hessian-aware Cholesky int6)
- **为什么要试**：比简单量化更精确
- **当时状态**：简单的 per-row symmetric int6
- **预估收益**：-0.0026 bpb
- **尝试前问**：
  - [ ] 实现复杂度高
  - [ ] 当前瓶颈是量化吗？

## 实验原则

1. **一次只改一个变量**：方便归因
2. **先问再做**：状态可能已变
3. **记录负面结果**：避免重复踩坑
4. **成本收益分析**：实现难度 vs 预期收益
