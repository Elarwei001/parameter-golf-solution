## 🔬 各环节详细探索

### 1️⃣ 数据预处理

| 模块 | 尝试 | 结果 | BPB 影响 | 状态 | 学习/Insight |
|------|------|------|----------|------|--------------|
| **Tokenizer** | 字节级 (256) | Baseline | 2.28 | ✅ 基准 | - |
| | BPE-1024 | **-23%** | 1.68 | ✅ 有效 | 更长 token = 更少序列 = 更快学习 |
| | **BPE-8192** | **-35%** ⭐⭐⭐ | **1.40** | ✅ **最佳** | 16MB 限制下的甜蜜点 |
| | BPE-32K | 未尝试 | - | ⏸️ | Embedding 太大会超限 |
| **数据过滤** | PPL 过滤 (P5-P95) | -0.17% | 1.52 | ❌ 负面 | 多样性 > 质量过滤 |
| | 加权采样 (σ=80) | -0.17% | 1.52 | ❌ 负面 | 同上 |
| | 加权采样 (σ=150) | -0.41% | 1.53 | ❌ 负面 | 放弃此方向 |
| **Embedding** | dim=512 | Baseline | - | ✅ 基准 | PR 分析: 训练后只用 20% 维度 |
| | dim=128 + 更多层 | +15.5% | 1.71 | ❌ 失败 | 训练需要大空间探索 |
| | dim=128 + Whitening | +15.2% | 1.70 | ❌ 失败 | Whitening 几乎没帮助 |
| | dim=256, 12层 | +6.5% | 1.58 | ❌ 一般 | 不如标准配置 |

**💡 核心发现**: Tokenizer 选择的影响 >>> 模型架构调整（35% vs 5%）

### 2️⃣ 模型架构

#### A. 基础架构选择

| 模块 | 尝试 | 结果 | BPB | 状态 | 学习/Insight |
|------|------|------|-----|------|--------------|
| **架构类型** | GPT (Transformer) | ✅ | 1.40 | ✅ 采用 | 稳定可靠 |
| | Mamba-3 (SSM) | 5.42 | ❌ 太慢 | ❌ 放弃 | 纯 PyTorch 不行，需要 Triton |
| | Text Diffusion | N/A | ❌ | ❌ 放弃 | BPB 评估不兼容 |
| **层数 × 维度** | 9层, dim=512 | 基准 | 1.40 | ✅ | 26.5M params |
| | 11层, dim=416 | **最佳** | **1.11** | ✅ **采用** | 层数 > 维度 |
| | 12层, dim=512 | -0.4% | 1.396 | ✅ 略好 | 38.8M, 还有空间 |
| | 9层, dim=480 | +0.6% | 1.41 | ❌ | 不如增加层数 |
| | Tapered (480→320) | +8% | 1.59 | ❌ 失败 | 后层 dim 太小限制能力 |

| 模块 | 尝试 | 结果 | BPB 影响 | 状态 | 学习/Insight |

| | 滑动窗口 (w=192) | -4.7% | 2.18 | ✅ 早期 | 中距离依赖甜蜜点 |
| | 滑动窗口 (w=64) | -3.0% | 2.21 | ❌ | 太小 |
| **Position Encoding** | RoPE (全维度) | 基准 | - | ✅ | - |

#### C. 激活函数

| 模块 | 尝试 | 结果 | BPB | 状态 | 学习/Insight |
|------|------|------|-----|------|--------------|
| **激活函数** | GELU | 基准 | 2.28 | ✅ 基准 | - |
| | **LeakyReLU(0.01)²** | **-4.3%** | **2.18** | ✅ **采用** | 平滑 + 非零负梯度 |

#### D. 残差连接 (mHC 系列) ⭐ 新发现

| 模块 | 尝试 | 结果 | BPB | 状态 | 学习/Insight |
|------|------|------|-----|------|--------------|
| **残差权重** | Standard (x + out) | 基准 | 1.5187 | ✅ 基准 | - |
| | mHC v1 (x + α·out) | **-0.7%** | 1.5293 | ❌ 更差 | 但 α 分布有价值！ |
| | **mHC v2 (α·x + β·out)** | **+0.13%** ⭐ | **1.5167** | ✅ 有效 | 揭示层级差异 |
| **渐进式 Attention** | 3 MLP-only + 8 full | 参数-6.5%, BPB-0.25% | 1.5225 | 🔬 验证 | 浅层 attn 确实不重要 |

**🔥 mHC v2 关键发现**:
- β_attn 峰值: Layer 7 (1.092) — 深层 Attention 很重要
- β_mlp 峰值: Layer 3 (0.801) — 浅层 MLP 主导
- **三阶段**: 浅层 MLP 主导 → 过渡区 → 深层 Attention 主导

## ❌ 失败清单 (重要!)

这些尝试没有带来收益，但提供了重要学习:

| 尝试 | 结果 | 为什么失败 | 学到什么 |
|------|------|-----------|----------|
| Muon Optimizer | 比 AdamW 差 | 短训练场景不适合 | 不要盲信论文 |
| Mamba-3 | 太慢 | 需要 Triton kernel | 纯 PyTorch SSM 不实用 |
| dim=128 + whitening | +15% | 训练需要大空间探索 | PR 低 ≠ 可以直接缩小 dim |
| PPL 数据过滤 | -0.2~0.4% | 多样性 > 质量 | 不要过度过滤 |
| EMA | -0.8% | 训练步数太少 | EMA 需要长训练 |
| mHC v1 | -0.7% | 单参数不够灵活 | 但揭示了层级差异 |
| Tapered dimension | +8% | 后层 dim 太小 | 不要在后层省参数 |

## 🔮 待探索地图

基于当前发现，高价值待探索方向:

```
                          优先级
                    高 ◄─────────► 低
               ┌─────────────────────────┐
         高    │ ① 更长训练     ② BigramHash │
         ↑     │ (5k→8k步)    (榜首在用)    │
      预 │     ├─────────────────────────┤
      期 │     │ ③ 层级差异化   ④ LeakyReLU │
      收 │     │ (基于mHC)    (0.01→0.5)  │
      益 │     ├─────────────────────────┤
         ↓     │ ⑤ Partial    ⑥ GPTQ      │
         低    │    RoPE      (复杂)       │
               └─────────────────────────┘
```

### 基于 mHC 发现的具体方向:

1. **层级差异化架构**
   - 浅层: 更小的 attention 或 MLP-only
   - 深层: 完整 attention 甚至更强
   - 省的参数给深层

2. **固定 α/β 训练**
   - 用 mHC v2 学到的 α/β 作为固定超参
   - 省掉学习开销，保留好处

3. **20 层深度实验** (进行中 🔄)
   - 观察峰值位置变化
   - 确认是否按比例移动
